# HubX

Мессенджер на протоколе XMPP. Свой сервер ejabberd, веб-клиент на React и Strophe.js, тонкий backend для регистрации и админ-панели. Сеть федеративная: пользователи разных серверов переписываются между собой так же, как по электронной почте.

## Как это устроено

Браузер подключается к ejabberd напрямую по WebSocket и говорит на XMPP (стэнзы `<message>`, `<presence>`, `<iq>`). Переписка идёт через XMPP-сервер, не через backend. Backend закрывает две задачи, которых нет в XMPP в готовом виде: регистрацию с ограничением частоты и каталог пользователей с поиском. Админ-панель обращается к backend по подписанному токену.

| Слой | Технология | Что делает |
|------|-----------|-----------|
| XMPP-сервер | ejabberd 24.12 (Docker) | Доставка сообщений, presence, ростер, оффлайн, история, файлы, группы, аутентификация SASL SCRAM |
| Веб-клиент | Vite + React + TypeScript + Strophe.js | Интерфейс. Подключается к серверу по WebSocket `ws://localhost:5280/ws` |
| Backend | Node + Express | Регистрация, поиск людей, админ-токены (HMAC), ссылки-приглашения в группы, web-push |

## Возможности

**Чаты.** Личные чаты в реальном времени, presence (в сети, отошёл, не беспокоить), индикатор «печатает». Галочки показывают статус: одна значит отправлено, две доставлено, синие прочитано. Есть ответы с цитатой, редактирование, пересылка с предпросмотром, реакции эмодзи. Сообщения можно удалить у себя или у всех, выбрать несколько штук сразу, очистить переписку целиком, заблокировать собеседника. Недописанный текст сохраняется при переключении между чатами.

**Группы.** Комнаты закрытые: попасть в чужую можно по приглашению от администратора или по ссылке-приглашению со сроком действия, который выбирает создатель. Администратор назначает и снимает других администраторов.

**Секретные чаты.** Сквозное шифрование на стороне клиента (ECDH P-256 + AES-GCM), сервер видит только шифртекст. Ключ собеседника подписывается долговременным ключом-удостоверением, поэтому подмену видно сразу. Отпечаток можно сверить вживую, вложения тоже шифруются. Сообщения уничтожаются по таймеру: у получателя отсчёт идёт с момента прочтения, а не с момента доставки.

**Файлы.** Фото показываются прямо в переписке, файлы приходят чипом с именем и размером, голосовые записываются с микрофона и проигрываются внутри сообщения с настоящей звуковой волной, в том числе у входящих (XEP-0363).

**История и профиль.** Переписка личных чатов и групп хранится на сервере (MAM) и подгружается при прокрутке вверх, плюс локальный кэш на устройстве. Реакции тоже сохраняются в архиве, поэтому переживают перезагрузку и приходят на новое устройство. У профиля (vCard) есть фото, которое видят собеседники. Людей можно найти через каталог сервера. Закреплённые сообщения, избранное и архив синхронизируются между устройствами (XEP-0049).

**Уведомления.** Звук, браузерные уведомления, когда окно не в фокусе, и web-push при полностью закрытой вкладке через service worker. Отдельно приходит уведомление, когда контакт появляется в сети. Статусы «отошёл» и «не беспокоить» выключают звук и всплывающие окна.

**Админ-панель.** Список пользователей, IP входа с городом и страной, разделение на активных и офлайн.

**Федерация.** Два независимых ejabberd-сервера соединены по s2s. Пользователь первого пишет пользователю второго по адресу `имя@домен-второго-сервера`, сообщение уходит между серверами напрямую.

**Интерфейс.** Тёмная и светлая темы, сменные обои чата, адаптив под телефон, планшет и десктоп.

## Запуск

Нужны Docker и Node 18+.

```bash
docker compose up -d                       # ejabberd (основной узел + второй для федерации)
cd server && npm install && npm start      # backend на http://localhost:4000
cd web && npm install && npm run dev        # клиент на http://localhost:5173
```

В Windows весь стек поднимается одной командой: `./scripts/start.ps1`.

Аккаунт администратора для панели создаётся так:

```bash
docker exec hubx-xmpp ejabberdctl register admin localhost 'ВашПароль'
```

Остальные регистрируются прямо в приложении. Локальный запуск подробно расписан в [РАЗВЁРТЫВАНИЕ.md](РАЗВЁРТЫВАНИЕ.md).

## Бот-ассистент

`hubx-bot` присылает уведомление о входе и отвечает на команды: погода по городу, конвертер валют и единиц, напоминания по времени (`/remind 15:30 текст`), `/help`. Запуск:

```bash
docker exec hubx-xmpp ejabberdctl register hubx-bot localhost 'ПарольБота'
cd examples/hubx-bot && npm install && node bot.js
```

## Стек и протоколы

ejabberd 24.12, Strophe.js 3 (в браузере и в Node для ботов через шим `ws` + `jsdom`), React 18 + Vite 5 + TypeScript. Использованные XEP: 0045 (группы), 0054 (vCard), 0184 (доставка), 0333 (прочтение), 0313 (история), 0363 (файлы), 0085 («печатает»), 0049 (приватное хранилище настроек), 0444 (реакции), 0461 (ответы), 0280 (carbons). Секретные чаты собраны на WebCrypto: ECDH P-256 и AES-GCM для шифрования, ECDSA для подписи ключей.

## Безопасность

Backend закрыт HMAC-токенами: пароль администратора проверяется в ejabberd, после чего выдаётся подписанный токен на 30 минут. Без него `GET /api/users` и `DELETE` отвечают `401`. Настроены CORS-аллоулист, helmet, ограничение частоты на регистрацию и логин, секреты через переменные окружения. Доступ к HTTP-API ejabberd сужен до минимального набора команд (`api_permissions`) и наружу не выходит: в проде Caddy проксирует только `/ws` и `/upload`. В облачном развёртывании Caddy добавляет `Strict-Transport-Security` и `Content-Security-Policy`, настроенный под реальные источники приложения. Переписка в секретных чатах шифруется на клиенте, сервер хранит только шифртекст. Подробный разбор с проверками собран в [SECURITY.md](SECURITY.md).
