# HubX: локальный запуск

Мессенджер работает на протоколе XMPP (сервер ejabberd). Веб-клиент написан на React и Strophe.js. Рядом стоит backend на Node/Express: он отвечает за регистрацию, поиск пользователей и админ-токены. Ниже четыре шага, чтобы поднять всё на своей машине. Облачный вариант описан в [РАЗВЁРТЫВАНИЕ-ОБЛАКО.md](РАЗВЁРТЫВАНИЕ-ОБЛАКО.md).

## Что понадобится

- Docker Desktop: https://www.docker.com/products/docker-desktop
- Node.js 18 или новее: https://nodejs.org

Проверка:
```bash
docker --version      # 24+
node --version        # v18+
```

## 1. XMPP-сервер

Из корня проекта:
```bash
docker compose up -d
```

Команда поднимает два узла ejabberd: основной и второй для федерации. Модули MUC (группы), HTTP Upload (файлы), MAM (история), vCard (профили) и s2s уже настроены в конфиге.

Проверка:
```bash
docker ps                                   # оба контейнера healthy
docker exec hubx-xmpp ejabberdctl status    # "started"
```

Клиент подключается по WebSocket: `ws://localhost:5280/ws`.

Порты: 5222 (c2s), 5269 (s2s), 5280 (HTTP, WebSocket, Upload), 5281 (второй узел).

## 2. Backend

```bash
cd server
npm install
npm start
```

Сервер слушает http://localhost:4000, ходит в ejabberd через его HTTP-API и отдаёт эндпоинты `/api/health`, `/api/register`, `/api/user-exists`, `/api/users/search`, `/api/users`, `/api/admin/login`. Для локальной работы переменные окружения задавать не нужно. На реальном сервере задайте `ADMIN_PASS`, `ADMIN_API_SECRET`, `ADMIN_USERS`.

## 3. Веб-клиент

```bash
cd web
npm install
npm run dev            # дев-сервер на http://localhost:5173
# или прод-сборка:
npm run build && npm run preview
```

Откройте http://localhost:5173 и зарегистрируйте аккаунт через вкладку «Регистрация». Чтобы проверить переписку, откройте второе окно или другой браузер, заведите второй аккаунт, найдите первый через поиск и подтвердите запрос.

Для админ-панели создайте администратора из консоли:
```bash
docker exec hubx-xmpp ejabberdctl register admin localhost 'YourStrongPass'
```
После входа под этим аккаунтом в разделе «Админ» виден список пользователей и кто из них онлайн.

## 4. Бот-ассистент (по желанию)

```bash
cd examples/hubx-bot
npm install
node bot.js
```

Бот логинится как `hubx-bot` и отвечает на команды `/help`, `/weth`, `/conv`, `/remind`.

## Архитектура

```
[Браузер: React + Strophe.js]  ──WebSocket(XMPP)──▶  [ejabberd]  ◀──s2s──▶  [ejabberd-2]
        │                                                 ▲
        └──HTTPS REST──▶ [Node/Express backend] ──HTTP-API─┘
```

Клиент говорит с ejabberd напрямую по XMPP поверх WebSocket. Backend закрывает то, чего у XMPP нет из коробки: регистрацию с rate-limit, поиск людей, проверку имён, админ-токены (HMAC). Серверы федерируются по s2s и работают как почта: пользователь одного сервера пишет пользователю другого.

## Если что-то не работает

- Docker перезапустился после сна компьютера. Выполните `docker compose up -d` ещё раз и перезапустите vite, backend и бота.
- Клиент не подключается. Проверьте, что `ws://localhost:5280/ws` доступен и контейнер в состоянии healthy (`docker ps`).
- Ошибки CORS у backend. По умолчанию разрешены только `http://localhost:5173` и `http://127.0.0.1:5173`. Другой адрес добавьте в переменную окружения `ALLOWED_ORIGINS` в `server/`.
- Файлы не загружаются. mod_http_upload отдаёт их по `http://localhost:5280/upload`, порт 5280 должен быть доступен из браузера.
